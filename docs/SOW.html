Project Overview

What we're building and why it matters

GharKaChef is a two-sided marketplace connecting verified home chefs with customers seeking authentic, home-cooked Indian meals across Canada. The platform combines two proven business models under one roof ‚Äî giving customers the flexibility to order on-demand or plan ahead, while giving chefs maximum control over their kitchen operations.

Core Value Proposition: Unlike restaurants, GharKaChef offers ghar ka taste (home taste) ‚Äî authentic regional Indian cuisine cooked in verified home kitchens, with full transparency on allergens, dietary options, and chef credentials.

Two Business Models, One Platform

‚ö°

Order Now ‚Äî The Uber Model

On-demand ordering. Chefs toggle online/offline like Uber drivers. Customers browse available chefs, pick from their live menu, and order with a 4-8 hour prep window. Chefs accept/decline, cook, and mark ready. Customers pick up at the scheduled time slot.

  • Chef goes online ‚Üí appears in browse
  • Customer orders ‚Üí chef accepts in real-time
  • 4h or 8h prep window with pickup slots
  • Daily capacity limits (like surge control)
  • Order status: Placed ‚Üí Accepted ‚Üí Cooking ‚Üí Ready
üìÖ

Pre-Order ‚Äî The Shef.com Model

Planned meal marketplace. Chefs schedule meals days in advance ‚Äî customers browse upcoming menus, pre-order before they're cooked, and subscribe to weekly plans. Limited drops create urgency with countdown timers and max order caps.

  • Chef plans meals for future dates
  • Customers pre-order before cooking starts
  • Limited Drops with countdown timers
  • Max order limits (e.g., "only 12 plates!")
  • Weekly meal plan subscriptions

How They Compare

Aspect Order Now (Uber Model) Pre-Order (Shef.com Model)
When Same-day, 4-8 hours from now Tomorrow, next week, scheduled dates
Chef Control Toggle online/offline, accept/decline per order Plan menu in advance, set max orders per meal
Customer Behavior "I'm hungry, who's cooking today?" "What's coming up this week? Let me plan."
Supply Model Reactive ‚Äî chef cooks when orders come in Proactive ‚Äî chef decides what to cook, customers opt in
Urgency Real-time availability (online/offline) Scarcity (limited drops, countdown, spots left)
Recurring Revenue Per-order transactions Weekly plan subscriptions + per-order
Inspired By Uber, DoorDash Shef.com, meal kit subscriptions

Why both? The Uber model captures impulse demand ("I want biryani tonight"), while the Shef.com model builds predictable revenue through subscriptions and pre-orders. Chefs benefit from both ‚Äî on-demand orders fill today's capacity, while planned meals guarantee tomorrow's income. Customers get the best of both worlds.

Technology Stack

Platform architecture and tools

Layer Technology Justification
Mobile App React Native (Expo SDK 54) Single codebase for iOS, Android, and Web. Fast iteration with Expo.
Language TypeScript Type safety, better IDE support, fewer runtime errors.
Navigation React Navigation (Tabs + Stack) Industry standard for React Native. Role-based conditional routing.
State Management React Context + useReducer Lightweight, no extra dependencies. 15 action types. localStorage persistence.
UI Design Glassmorphism (backdrop-filter, expo-blur, gradients) Modern, premium feel. Frosted glass cards with emoji icons.
Backend (Phase 2) Node.js / Next.js API Routes Full-stack JavaScript. Server-side rendering for web. API routes.
Database (Phase 2) PostgreSQL + Prisma Relational data (chefs, menus, orders). Type-safe ORM.
Payments (Phase 2) Stripe Industry standard. Supports marketplace payouts (Connect).
Hosting Vercel (Web) + EAS (Mobile) Zero-config deployment. Automatic preview URLs. CDN.
